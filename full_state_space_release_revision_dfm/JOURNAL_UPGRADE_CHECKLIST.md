# Journal Upgrade Checklist

This checklist defines what must be true before the project can claim a full joint Kalman/EM release-revision DFM in a journal manuscript.

## Implemented In This Folder

- Generic Kalman filter with arbitrary missing observations.
- Rauch--Tung--Striebel smoother.
- EM loop for linear-Gaussian models.
- Release-structured GDP block for `A/S/T/M`.
- GDP revision state with ordered release-revision loadings.
- Joint monthly-indicator revision state using first-vintage and mature-vintage indicator measurements.
- Gaussian density utilities for intervals, log score, CRPS, and coverage.
- Synthetic smoke tests.
- Bronze-data adapter and bronze smoke example.
- Quarterly prototype backtest runner that produces point forecast, revision forecast, metric, and failure CSVs.
- Prototype leakage controls for current-quarter release masking and mature-value maturity-lag masking.
- Exact/pseudo model-family backtest runner comparing `ar`, `bridge`, `standard_dfm`, `release_dfm`, `revision_dfm_kalman_em`, and `joint_indicator_revision_dfm_full_kalman_em`.
- Forecast, revision, metric, exact-minus-pseudo gap, and failure CSVs for the exact/pseudo runner.
- GDP release calendar builder from ALFRED `GDPC1` vintage dates, with full vintage-derived A/S/T coverage for the 2005Q1--2024Q4 headline sample.

## Required Before Journal Claims

- Cross-check ALFRED-derived GDP release dates against official BEA news-release archive pages where available.
- At each forecast origin, fit the model using only information available at that origin.
- Generate forecasts for `A`, `S`, `T`, `DELTA_SA`, and `DELTA_TS`.
- Save forecasts with the same schema as existing benchmark outputs.
- Recompute RMSE, MAE, bias, sign accuracy, exact-vs-pseudo gaps, and small-sample DM tests.
- Add density metrics: log score, CRPS, 68/90 percent interval coverage.
- Compare full joint Kalman/EM against the current factor-regression release/revision models on the full 2005Q1--2024Q4 sample with a publication-quality EM iteration cap.
- Freeze a new result folder before editing the manuscript.

## Claims Allowed After Backtest Passes

- "The model is estimated as a joint state-space system by Kalman filtering, smoothing, and EM."
- "GDP release revisions and selected monthly indicator revisions are represented in the same latent-state system."
- "Forecast densities are evaluated in addition to point forecasts."

## Claims Not Allowed Until Then

- Do not say the existing final-project headline numbers were generated by full Kalman/EM.
- Do not say the joint indicator-revision model improves accuracy until a rolling backtest proves it.
- Do not compare density scores until they are generated from the same real-time forecast origins.
